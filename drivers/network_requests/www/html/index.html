Variable tokens

            
PARAM{deviceId,variableId} is replaced with the variable's current
            value each time the request is sent. Click a variable in the browser to insert its
            token into the focused field. Variables can also be referenced by name:
            PARAM{96,HUMIDITY} (exact match, case sensitive). Tokens work in the URL,
            body, header values, and Text payloads.

          

          

            
Text payloads (TCP / UDP)

            
Sent as typed, after decoding the escapes \r \n
            \t \0 \\ and \xNN (two hex
            digits). For example, the payload PWR ON\r\n sends
            PWR ON followed by a carriage return and a line feed.

          

          

            
Hex payloads (TCP / UDP)

            
To send raw bytes instead of text, switch the Text | Hex toggle above the
            payload field to Hex, then write the whole payload as hex digits. Whitespace, commas,
            and 0x prefixes are ignored, so 02 41 54 0d,
            0x02,0x41,0x54,0x0d, and 0241540d all send the same 4
            bytes. Variable tokens are not substituted in Hex payloads.

          

          

            
HTTP

            
HTTPS is supported; TLS certificates are only checked when
            Validate TLS certificate is on, so self-signed LAN devices work by default.
            Basic auth goes in the URL: http://user:pass@host/path. The last response
            body is published to the <Name> Response variable.
